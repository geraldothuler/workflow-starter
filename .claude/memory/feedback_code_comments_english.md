All code comments (inline, block, SQL, Kotlin, Go, TypeScript, etc.) must be written in English — in every repository, without exception.

**Why:** Standard engineering practice; keeps codebases consistent and accessible to all contributors.

**How to apply:** Any time writing or editing code in any Cobli repo (trigger-action-api, fusca, iris, herbie-dashboard, webhook, janus, etc.), write every comment in English. This applies to: `//`, `/* */`, `--` (SQL), `#` (scripts), JSDoc, KDoc — everything. The conversation with the user stays in pt-BR; only the code artifacts switch to English.
